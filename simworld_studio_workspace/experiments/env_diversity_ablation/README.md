# Environment Diversity Ablation Study

**Research question:** Does training on more diverse environments improve downstream PointNav agent performance?

## Design

- **17 unique UE maps** generated by coding agents (15 train, 2 test)
- **Conditions:** 1 / 5 / 10 / 15 training scenes (total training episodes fixed at 30)
- **Test set:** Fixed across all conditions (2 held-out maps + 1 held-out episode per training map = 31 episodes)
- **Memory:** Strategy memory (episode-level LLM reflection, accumulates across episodes)
- **Metrics:** SR, SPL, SoftSPL

## Quick Start

All commands run from `simworld_studio_workspace/`.

### 1. Copy maps into UE project

```bash
python -m experiments.env_diversity_ablation.prepare_maps
```

Copies 17 `.umap` files from `valid_umaps/` into your UE project's `Content/AblationMaps/`.

**Then restart UE** (or use `--map /Game/AblationMaps/ablation_06` to launch directly on an ablation map).

### 2. Generate tasks (requires running UE)

```bash
python -m experiments.env_diversity_ablation.generate_tasks \
    --mcp-port 55558 --ucv-port 9002
```

Samples PointNav episodes via NavMesh for each map. Retry a single map with `--map-index N`.

### 3. Compose train/test conditions

```bash
python -m experiments.env_diversity_ablation.compose_conditions
```

Offline step — creates `conditions/{1,5,10,15}_scenes/train_episodes.json` and `conditions/test_episodes.json`.

### 4. Run the ablation (requires running UE + vLLM)

```bash
# Dry run
python -m experiments.env_diversity_ablation.run_ablation --dry-run

# Run specific conditions (1 epoch, with resume)
python -m experiments.env_diversity_ablation.run_ablation \
    --condition 1 15 --epochs 1 --resume \
    --mcp-port 55558 --ucv-port 9002
```

**Key flags:**
- `--resume`: Skip already-completed episodes (checks `summary.json` files)
- `--condition N [M ...]`: Which conditions to run
- `--epochs K`: Number of training epochs (default 2)

**Parallel execution:** Run different conditions on separate UE instances:
```bash
# Instance 1: GPU 3, condition 1
python -m experiments.env_diversity_ablation.run_ablation \
    --condition 1 --epochs 1 --resume --mcp-port 55558 --ucv-port 9002

# Instance 2: GPU 4, condition 15
python -m experiments.env_diversity_ablation.run_ablation \
    --condition 15 --epochs 1 --resume --mcp-port 55559 --ucv-port 9003
```

### 5. Switching models

Edit `config.py`:
```python
LLM_MODEL = "qwen"             # or "claude", "gpt", "gemini"
LLM_MODEL_ID = "Qwen3.5-27B"   # model name on vLLM
LLM_BASE_URL = "http://your-server:8001/v1"
```

For models that don't support OpenAI tool_calls natively (e.g. local vLLM), the runner automatically uses `text_action_mode=True` which parses actions from the LLM's text output.

## Config Reference (`config.py`)

| Parameter | Default | Notes |
|-----------|---------|-------|
| `MCP_PORT` | 55558 | UE MCP port |
| `UCV_PORT` | 9002 | UnrealCV port |
| `LLM_BASE_URL` | `http://132.239.95.133:8001/v1` | vLLM server |
| `LLM_MODEL_ID` | `Qwen3.5-27B` | Model name |
| `MIN_GEODESIC_CM` | 300 | Min path length for episodes |
| `MAX_GEODESIC_CM` | 6000 | Max path length |
| `TRAIN_BUDGET` | 30 | Total training episodes per condition |
| `MAX_STEPS` | 40 | Max steps per episode |
| `MEMORY_BACKEND` | strategy | Memory type |

## Output Structure

```
results/ablation_{N}scenes/
  meta.json                          # run config
  epoch0_train_map{XX}/w{W}/         # per-wave episode logs
    ep_{I}_{agent}/
      summary.json                   # SR, SPL, SoftSPL, reward, steps
      episode.jsonl                  # per-step log
      llm_raw.jsonl                  # raw LLM responses
  epoch0_test_map{XX}/w{W}/          # test episodes
  epoch0_summary.json                # aggregated train + test metrics
  final_results.json                 # all epochs
```
