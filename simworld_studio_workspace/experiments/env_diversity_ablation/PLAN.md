# Environment Diversity Ablation Study — Execution Plan

## Goal

Validate whether training on more diverse environments improves downstream
PointNav agent performance. We have 17 unique UE maps generated by coding
agents. We hold out 2 for testing, use 15 for training, and ablate over
the number of distinct scenes used during training (1 / 5 / 10 / 15)
while keeping total training episodes fixed at 30.

## Experiment Design

| Condition   | # Scenes | Tasks/Scene | Total Train |
|-------------|----------|-------------|-------------|
| 1_scenes    | 1        | 30          | 30          |
| 5_scenes    | 5        | 6           | 30          |
| 10_scenes   | 10       | 3           | 30          |
| 15_scenes   | 15       | 2           | 30          |

- **Test set**: fixed across all conditions — 2 held-out maps + 1 held-out
  episode per training map
- **Model**: Qwen3.5-27B @ `http://132.239.95.133:8001/v1`
- **Memory**: strategy (accumulates across episodes within an epoch)
- **Epochs**: 2 per condition
- **Metrics**: SR, SPL, SoftSPL on the frozen test set

## Execution Steps

All commands run from:
```bash
cd /home/koe/SimWorld-Studio-Internal/simworld_studio_workspace
```

### Step 1: Copy maps into UE project

```bash
python -m experiments.env_diversity_ablation.prepare_maps
```

This copies 17 `.umap` files from `/home/koe/valid_umaps/` into
`/data/koe/simworld_studio_projects/Content/AblationMaps/ablation_00.umap` … `ablation_16.umap`.

**After this step, restart UE** so it detects the new Content files.
Launch UE with:
```bash
export UE_ROOT=/data/koe/UE_5.3.2
export UE_PROJECT_PATH=/data/koe/simworld_studio_projects

simworld-studio start \
  --data-dir ~/SimWorld-Studio-Internal/simworld_studio_workspace \
  --gpu 0 \
  --port 3002 \
  --mcp-port 55557
```

Wait for UE to fully start (check logs, ~60-90 seconds).

### Step 2: Generate tasks per map

```bash
python -m experiments.env_diversity_ablation.generate_tasks
```

For each of the 17 maps, the script:
1. Stops PIE
2. Loads the map via MCP: `unreal.EditorLoadingAndSavingUtils.load_map('/Game/AblationMaps/ablation_XX')`
3. Starts PIE (waits 12s for NavMesh plugin)
4. Builds NavMesh via UnrealCV: `vset /nav/build`
5. Samples episodes via `sample_pointnav_episode_navmesh()`
   - Training maps: 32 episodes each (need 30 for worst case + 1 held-out)
   - Test maps: 8 episodes each
6. Saves to `experiments/env_diversity_ablation/tasks/ablation_XX.json`

**If a map fails** (e.g., NavMesh too small), retry individually:
```bash
python -m experiments.env_diversity_ablation.generate_tasks --map-index 5
```

**Expected output**: 17 JSON files in `tasks/`, total ~500+ episodes.

**Potential issues**:
- Small maps (15-16 objects) may have limited navigable area → fewer valid
  episodes. The script handles this gracefully (logs warnings, saves what
  it gets).
- NavMesh build may fail on very small scenes → retry or lower
  `MIN_GEODESIC_CM` in config.py (default 800cm).
- If `load_map` doesn't work (returns MAP_LOAD_FAILED), the .umap may not
  have been detected by UE. Verify files exist in Content/AblationMaps/
  and restart UE.

### Step 3: Compose train/test conditions

```bash
python -m experiments.env_diversity_ablation.compose_conditions
```

Reads the per-map task files and creates:
- `conditions/test_episodes.json` — fixed test set (2 held-out maps + 1 ep per training map)
- `conditions/1_scenes/train_episodes.json` — 30 episodes from 1 map
- `conditions/5_scenes/train_episodes.json` — 6 eps × 5 maps
- `conditions/10_scenes/train_episodes.json` — 3 eps × 10 maps
- `conditions/15_scenes/train_episodes.json` — 2 eps × 15 maps

This step is offline (no UE needed). Check the output to verify allocations
look reasonable.

### Step 4: Run the ablation

**Dry run first** (no UE needed):
```bash
python -m experiments.env_diversity_ablation.run_ablation --dry-run
```

**Full run** (UE must be running):
```bash
python -m experiments.env_diversity_ablation.run_ablation \
    --condition 1 5 10 15 --epochs 2
```

Or run conditions one at a time (safer, can restart between):
```bash
python -m experiments.env_diversity_ablation.run_ablation --condition 1 --epochs 2
python -m experiments.env_diversity_ablation.run_ablation --condition 5 --epochs 2
python -m experiments.env_diversity_ablation.run_ablation --condition 10 --epochs 2
python -m experiments.env_diversity_ablation.run_ablation --condition 15 --epochs 2
```

For each condition, the runner:
1. Groups training episodes by source map
2. For each source map: loads it in UE → runs its episodes (memory=strategy, train mode)
3. After all training episodes: loads each test map → runs test episodes
   (memory=ReadOnly, frozen from training)
4. Logs SR / SPL / SoftSPL per epoch

**Results** go to `experiments/env_diversity_ablation/results/ablation_Nscenes_TIMESTAMP/`:
- `meta.json` — run config
- `epoch0_summary.json`, `epoch1_summary.json` — per-epoch metrics
- `final_results.json` — all epochs aggregated

After all 4 conditions complete, the runner prints a summary table:

## Config Reference

Key parameters in `config.py` that you might need to adjust:

| Parameter | Default | Notes |
|-----------|---------|-------|
| `MCP_PORT` | 55557 | Your assigned MCP port |
| `UCV_PORT` | 9001 | Your assigned UnrealCV port |
| `LLM_BASE_URL` | `http://132.239.95.133:8001/v1` | Qwen3.5-27B vLLM server |
| `LLM_MODEL_ID` | `Qwen3.5-27B` | Model name on vLLM |
| `TASKS_PER_MAP` | 32 | Episodes to sample per training map |
| `MIN_GEODESIC_CM` | 800 | Min path length — lower if small maps fail |
| `MAX_GEODESIC_CM` | 4000 | Max path length |
| `TRAIN_BUDGET` | 30 | Total training episodes per condition |
| `N_EPOCHS` | 2 | Training epochs |
| `MEMORY_BACKEND` | strategy | Memory type for agent |
| `MAX_STEPS` | 40 | Max steps per episode |

## Estimated Runtime

- Step 1: ~5 seconds (file copy)
- Step 2: ~30-60 minutes (17 maps × ~2 min each for load + navmesh + sampling)
- Step 3: ~5 seconds (offline composition)
- Step 4: ~4-8 hours total (4 conditions × 2 epochs × ~30 episodes × ~40 steps × LLM latency)
  - Each condition: ~1-2 hours
  - Can run conditions sequentially or one at a time with breaks
